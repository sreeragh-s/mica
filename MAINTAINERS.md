# Maintainers

This file documents expected maintainer responsibilities for NoteLab.

## Responsibilities

Maintainers are expected to:

- triage issues and pull requests
- review security-sensitive changes carefully
- keep release notes and contributor documentation current
- protect user data and local filesystem safety as first-order concerns
- enforce the [Code of Conduct](./CODE_OF_CONDUCT.md)

## Decision making

For small changes, approval from one maintainer is enough. For broad
architecture changes, security-sensitive behavior, license changes, or release
process changes, maintainers should seek consensus before merging.

## Becoming a maintainer

Regular contributors may be invited to maintainership after they demonstrate:

- sustained, constructive participation
- sound judgment around user data and native desktop permissions
- consistent review quality
- respect for project scope and community norms

Maintainer access should be reviewed periodically and removed when it is no
longer needed.
