# Governance

NoteLab is maintained by the project maintainers listed or delegated through
repository permissions. Governance is intentionally lightweight while the
project is early.

## Scope

The project prioritizes a local-first desktop notes workflow with careful
handling of user files, Git operations, native permissions, and optional AI
integrations.

## Decisions

Routine fixes and documentation improvements can be merged after normal review.
Changes that affect project direction should be discussed in an issue or pull
request before implementation.

Examples of direction-setting changes:

- license or governance changes
- workspace file format changes
- persistent data migrations
- new networked services or telemetry
- native permission changes
- release and distribution changes
