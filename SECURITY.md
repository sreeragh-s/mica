# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do NOT** create a public GitHub issue for security vulnerabilities.

### How to Report

1. **Email**: Send details to the maintainer directly (you can find contact info on GitHub)
2. **Private Security Advisory**: Use GitHub's [Private Vulnerability Reporting](https://github.com/sreeraghs/notelab/security/advisories/new) feature

### What to Include

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Any suggested fixes (optional)

### Response Timeline

- We aim to acknowledge reports within 48 hours
- We will provide updates on the resolution timeline
- We appreciate patience as we work on a fix

### Scope

This policy applies to:

- The notelab application
- All official plugins and extensions
- Documentation and build scripts

Thank you for helping keep notelab secure!
