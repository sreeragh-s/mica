# Support

## Getting help

Use GitHub issues for reproducible bugs and well-scoped feature requests.
Before opening an issue:

- search existing issues and discussions
- check [README.md](./README.md) and [docs/ARCHITECTURE.md](./docs/ARCHITECTURE.md)
- include platform details, app version or commit, and relevant logs

## Questions

For usage or development questions, open a discussion if the repository has
GitHub Discussions enabled. If discussions are unavailable, use a GitHub issue
and label it as a question.

## Security reports

Do not use public issues for vulnerabilities. Follow [SECURITY.md](./SECURITY.md).

## Maintainer availability

This is an open-source project maintained as capacity allows. Clear reproduction
steps, screenshots, logs, and small pull requests make support much easier.
